Description

As discussed in this thread:
http://thread.gmane.org/gmane.comp.java.stripes.user/4612

I suggest adding a boolean attribute, 'enabled', to the stripes:link tag. This attribute would be optional and true by default. If set to false, the tag renders the body but not the link, such that the link appears as plain text only. This is useful, for example, to enable links based on some dynamic condition, e.g. user rights.

<stripes:link href="..." enabled="${user.allowed}">Link</stripes:link>

Very easy to implement, for example, against SVN trunk version 608:

Gregg Bolinger added a comment - 02/Oct/07 9:54 AM

My suggestion is to add 2 attributes, both optional.

enabled - used as described above. I'd never see myself using this so I cannot comment on span vs a vs bla bla.

rendered - either the link shows up or it doesn't. I've already implemented this myself in the LinkTag.java the following way:

@Override
public int doEndTag() throws JspException {
if (isRendered()) {
try {
set("href", buildUrl());
writeOpenTag(getPageContext().getOut(), "a");
String body = getBodyContentAsString();
if (body != null) { getPageContext().getOut().write(body.trim()); }
writeCloseTag(getPageContext().getOut(), "a");
}
catch (IOException ioe) { throw new StripesJspException("IOException while writing output in LinkTag.", ioe); }
}
// Restore state and go on with the page
getAttributes().remove("href");
clearParameters();
return EVAL_PAGE;
}

there is a boolean rendered attribute in the LinkTagSupport class and I added the attribute to the tld file. There may be a better place to check for isRendered() but this worked fine for me. So I much prefer:

<stripes:link rendered="${ actionBean.someBooleanCheck }" ...>Link</stripes:link>

over

<c:if test="${ actionBean.someBooleanCheck }">
<stripes:link ...>Link</stripes:link>
</c:if>

And then imagine the nastiness that ensues with a c:choose/c:when.

Gregg
